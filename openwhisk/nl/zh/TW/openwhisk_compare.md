# 函數即服務比較
{: #openwhisk_faas_compared}

無伺服器架構並非所有計算問題的萬能措施，但確實可以解決一些問題。無伺服器設計可能是建議選擇時，有[許多使用案例](./openwhisk_use_cases.html)。我們將在這裡比較下列架構：

1. **函數即服務 (FaaS)** - 受管理 OpenWhisk。目前，IBM 是唯一提供受管理 [OpenWhisk on Bluemix](https://console.ng.bluemix.net/openwhisk) 的供應商。

2. 含「OpenWhisk 自行開發 (RYO)」的**基礎架構即服務 (IaaS)**。一般使用者可以從 Apache Incubation Project 下載 OpenWhisk，並在 [Bluemix IaaS](https://console.ng.bluemix.net/catalog/?category=devices) 或其他 [IaaS 雲端](https://en.wikipedia.org/wiki/Cloud_computing#Infrastructure_as_a_service_.28IaaS.29)上安裝並執行它。

3. **平台即服務 (PaaS)** - 受管理應用程式運行環境。建議範例是 IBM Bluemix CloudFoundry 實作所管理的 [Liberty for Java](https://console.ng.bluemix.net/catalog/starters/liberty-for-java) 運行環境。

4. **容器即服務 (CaaS)** - 受管理容器環境。建議範例是 IBM 的 [Containers on Bluemix](https://console.ng.bluemix.net/catalog/?category=containerImages)。

5. 含 Java EE 運行環境的**基礎架構即服務 (IaaS)**。建議範例是 IBM 的 [WebSphere Application Server VM on Bluemix](https://console.ng.bluemix.net/catalog/services/websphere-application-server)。

以下是在這些不同運行環境上開發及操作應用程式的**一般使用者視景**中，每一個架構選項的優缺點摘要：
